# Comparison (비교)

## 비교 대상

- Entity Server (현재 프로젝트)
- 전통 ORM 중심 CRUD 서버
- BaaS/Low-code 데이터 플랫폼

## 기능 비교

| 항목                  | Entity Server | ORM CRUD 서버                | BaaS/Low-code |
| --------------------- | ------------- | ---------------------------- | ------------- |
| 설정 기반 스키마 운영 | 강함          | 약함(코드 마이그레이션 중심) | 중간          |
| 동적 엔티티 확장      | 강함          | 중간                         | 강함          |
| 온프레미스 제어       | 강함          | 강함                         | 제한적        |
| 커스텀 비즈니스 로직  | 강함          | 강함                         | 중간          |
| 초기 학습 비용        | 중간          | 낮음                         | 낮음          |

## 장단점

### 장점

- 설정 중심으로 확장 속도가 빠름
- 운영 스크립트가 표준화돼 반복 작업이 단순함
- 엔티티별 히스토리/캐시/시드 정책 분리가 쉬움

### 단점

- SQL 방언/DDL 의존도가 있어 MongoDB, RedisJSON 같은 비SQL 스토리지는 미지원
- 잘못된 설정 배포 시 영향 범위가 넓을 수 있음
- 대규모 데이터에서는 인덱스 백필/초기화 전략이 필수

## 어떤 팀에 적합한가

| 상황                                         | 적합도 |
| -------------------------------------------- | ------ |
| 다양한 업무 테이블을 빠르게 추가/변경해야 함 | 높음   |
| 명확한 정적 도메인 + 복잡한 트랜잭션만 필요  | 중간   |
| 완전 관리형(BaaS) 선호, 서버 운영 최소화     | 낮음   |

## 선택 가이드

- 빠른 엔티티 확장 + 자체 운영 제어가 중요하면 Entity Server 권장
- DB 종속성 최소화가 최우선이면 SQL 추상화 레이어를 먼저 설계
